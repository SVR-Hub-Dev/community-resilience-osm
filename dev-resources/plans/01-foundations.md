## Approach

- Pin exact versions found via `npm view <pkg> version` at install time; commit the lockfile change.
- `drizzle.config.ts` is a Node-only file consumed by the `drizzle-kit` CLI; safe to read `process.env.DATABASE_URL` there. The Worker code reads its connection string from `platform.env` / `$env/dynamic/private`, never `process.env`.
- `db.ts` exports a **factory**, not a singleton. The CLAUDE.md rule "Database clients are request-scoped" is non-negotiable ‚Äî module-scope reuse across requests is the most common Workers footgun. Signature: `export function getDb(env: { DATABASE_URL: string }) { return drizzle(neon(env.DATABASE_URL)); }`. Callers pass `platform.env` from a `+server.ts` or `+page.server.ts`. No `schema` arg yet ‚Äî added in 02 once `schema.ts` exists.
- `.dev.vars.example` is committed; `.dev.vars` is ignored. Copying the example is a one-line README step (out of scope here, captured in follow-ups).
